DEEL 1 — WAT IS ER GEDAAN?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

VOOR: Klaar was een lokale demo. Elke gebruiker moest zijn eigen Anthropic
API key invoeren. Geen accounts, geen server-side logica. Onverkoopbaar als SaaS.

NA: Klaar is een echte SaaS-applicatie. Dit is er gebouwd:

1. SUPABASE DATABASE
   → 5 tabellen: profiles, recipes, ingredients, haccp_logs, api_usage
   → Row Level Security: elke gebruiker ziet alleen zijn eigen data
   → Automatische triggers: profiel aanmaken bij registratie

2. SUPABASE AUTH (EMAIL OTP)
   → Geen wachtwoorden — email in + 6-cijferige code = ingelogd
   → Werkt op file://, lokaal, en via Vercel
   → index.html heeft 2-stap login modal (email → code → redirect naar Module 01)
   → Alle 6 modules redirecten naar login als niet ingelogd

3. ANTHROPIC AI PROXY (EDGE FUNCTION)
   → Jouw Anthropic API key staat server-side (niemand ziet hem)
   → Alle AI-calls: Browser → Supabase Edge Function → Anthropic
   → Edge Function checkt of je ingelogd bent voor hij iets doorstuurt
   → Streaming werkt (chat-widgets tonen tekst real-time)
   → Token-gebruik opgeslagen per gebruiker (voor billing later)

4. ALLE 6 MODULES BIJGEWERKT
   → index.html + Module 01, 02, 03, 04, 07, 10
   → 0 directe Anthropic API calls meer in de code
   → klaar-client.js: gedeelde bibliotheek die elke module inlaadt

5. MODULE 05: MENUBUILDER (nieuw gebouwd)
   → Gebaseerd op uitgebreide menu engineering research
   → Laadt recepten en kostprijzen direct vanuit Module 01
   → Menu psychology: golden triangle, decoy pricing, anchor pricing
   → Live menu preview + AI assistent voor aanpassingen
   → Export als PDF of print-ready versie

DEEL 2 — HOE WERKT DE ARCHITECTUUR?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GEBRUIKER OPENT KLAAR
    ↓ Login met email + OTP code (Supabase Auth)
    ↓ Sessie-token opgeslagen in browser
    ↓ Module opent → checkt of token geldig is
    ↓ Gebruiker gebruikt AI-feature
    ↓ Browser stuurt verzoek naar Supabase Edge Function (met sessie-token)
    ↓ Edge Function checkt token → stuurt door naar Anthropic (jouw API key)
    ↓ Antwoord logt gebruik → terug naar browser

WAAROM DIT BETER IS:
- Gebruiker hoeft geen Anthropic account aan te maken
- Jij betaalt Anthropic, jij rekent door aan de klant
- API key onzichtbaar voor iedereen
- Code kopiëren werkt niet (geen geldige Supabase sessie)

EERLIJK: HTML/CSS/JS is leesbaar via DevTools. Onvermijdelijk bij web-apps.
Jouw echte bescherming: server-side logica, klantdata (lock-in), en snelheid
waarmee je nieuwe features bouwt. Concurrent kan de UI kopiëren maar niet
de backend, niet de data, niet het klantvertrouwen.
